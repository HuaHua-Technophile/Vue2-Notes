• vue-router是Vue.js官方的路由插件，它和vue.js是深度集成的，适合用于构建单页面应用。
  • vue的单页面应用是基于路由和组件的，**路由用于设定访问路径，并将路径和组件映射起来**。
  • 传统的页面应用，是用一些超链接来实现页面切换和跳转的。
  • 在vue-router单页面应用中，则是路径之间的切换，也就是组件的切换。
  • 路由模块的本质就是建立起url和页面之间的映射关系。
    vue路由的用法有两种模式：
  • hash模式：依靠hash值（锚链接）的变化进行实现，hash的变化会导致浏览器记录访问历史的变化、但是hash的变化不会触发新的URL请求。这种模式兼容性好，但是url中会有#号。
  • history模式：利用HTML5的history API（pushState和replaceState）来修改url地址栏，而不刷新界面。这种模式url更美观，但是需要服务器端配合重定向。
    vue路由还有两种导航方式：
  • 编程式导航：通过编写代码来实现路由跳转，比如使用router.push或router.replace方法。
  • 组件式导航：通过使用<router-link>组件来实现路由跳转，通过to属性指定目标地址，默认渲染为带有正确连接的<a>标签。
    vue路由还可以传递参数：
  • params方式：在定义路由时使用:name占位符，然后在跳转时传入params对象，比如{ name: 'user', params: { userId: 123 }}。这种方式不会改变url地址栏，但是需要在<router-view>组件上使用props属性接收参数。
  • query方式：在定义路由时不需要占位符，然后在跳转时传入query对象，比如{ path: '/user', query: { userId: 123 }}。这种方式会改变url地址栏，并且可以通过this.$route.query获取参数。
    vue路由的优点有：
  • 可以实现单页面应用，提高用户体验和开发效率。
  • 可以利用vue-router提供的API和组件来灵活地控制路由跳转和参数传递。
  • 可以利用路由守卫来实现权限控制和导航钩子。
    vue路由的缺点有：
  • 需要额外学习vue-router的知识和配置。
  • 首次加载会比较慢，因为需要加载所有组件。
  • 不利于SEO，因为搜索引擎爬虫无法抓取动态渲染的内容。

router-link是一个特殊的组件，会被转译为实体标签.不设置tag属性时,默认为a标签.该组件用于设置一个导航链接，切换不同 HTML 内容。
to属性为目标地址， 即要显示的内容
点击后,内部会立刻把 to 的值传到 router.push()，所以这个值可以是一个字符串或者是描述目标位置的对象

被激活的路由切换组件(router-link),会加上样式class="router-link-exact-active router-link-active"
